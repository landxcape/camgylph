# Scope: v1.4.0 Release Mode

- Status: Complete
- Date: 2026-03-02
- Milestone tag: `v1.4.0`

## Goal

Add a production-oriented release mode preset with predictable CLI precedence and clear runtime visibility.

## Completed

- Added `--release-mode` CLI flag.
- Added release-mode preset values:
  - `color_mode=ansi256`
  - `ramp=standard`
  - `show_metrics=false`
  - `render_fps=60`
  - `max_cols=0`, `max_rows=0`
- Implemented deterministic precedence:
  - Explicit flags override preset values regardless of argument order.
- Added startup profile summary to stderr when release mode is active.
- Updated `--help` text and README release-mode documentation.
- Added config unit tests for release-mode behavior and precedence.

## Why This Scope

- Provides a stable user-facing mode for smooth terminal rendering.
- Separates product defaults from development tuning.
- Makes scripted usage reliable due to order-independent precedence.

## Why Not More in This Scope

- No capture backend changes: keep performance baseline comparable.
- No packaging automation changes: release-mode behavior needed first.
- No new rendering effects: avoid mixing UX tuning with algorithm changes.

## Validation

```bash
cargo fmt
```

Expected:

- All tests pass.
- `--release-mode` enables preset defaults.
- Explicit flags still win when provided in any order.
