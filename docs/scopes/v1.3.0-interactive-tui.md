# Scope: v1.3.0 Interactive TUI Controls

- Status: Complete
- Date: 2026-03-02
- Milestone tag: `v1.3.0`

## Goal

Make the terminal output behave like a camera TUI while keeping runtime controls responsive and discoverable.

## Completed

- Added shortcuts/footer visibility toggle (`h`), default visible.
- Added horizontal mirror toggle (`v`) for selfie-style preview.
- Kept full-terminal `cover` render with aspect-preserving centered crop.
- Preserved live tuning controls (`m`, `c`, `r`, `+/-`, `[]`).
- Updated help output and README controls.
- Added mapper unit test for mirror behavior.
- Fixed renderer cache edge case when hiding footer.

## Why This Scope

- It directly improves live usability and camera-like behavior.
- It is a strong learning scope for terminal rendering state, input handling, and frame mapping.
- It de-risks UX before moving to packaging/performance release work.

## Why Not Bigger Changes Here

- No new capture backend was added to keep this milestone focused on runtime UX.
- No new codec/pixel pipeline work was added to avoid coupling UX and low-level capture changes.
- No release-only flags were added yet to avoid mixing implementation with next-scope planning.

## Validation

```bash
cargo fmt
cargo check
cargo test
```

Expected:

- Tests pass.
- Runtime controls work:
  - `h`: show/hide footer line
  - `v`: mirror on/off
  - `m`: metrics in footer on/off
